TAGS OHNE INHALT

        Die meisten HTML-Tags rendern irgendwelchen Inhalt.
        Aber es gibt auch welche, die zum anderen Zweck da sind.
        Tag `hr` z.B. setzt einfach eine horizontale Linie.
        Solche Tags haben kein (Start-/End-)Paar.
        In HTML4 hatten sie den Schrägstrich in ihrer Syntax.
        In HTML5 sehen sie so aus wie die Starttags.

TAGS MIT INHALT

        Die Tag-Paare bestehen deswegen aus zwei Teilen, 
        um die Grenzen für ihren Inhalt zu definieren.
        Dieser Inhalt wird (mit einpaar Ausnahmen) auch im Browser gerendert.
        Analysiere diese Datei. Die meisten Tags rendern Text.

VERSCHACHTELUNG

        Tags können weitere Tags als Inhalt in sich haben. 
        Analysiere dafür diese HTML-Datei.
        Wenn HTML-Elemente verschachtelt sind, 
        spricht man von Eltern-Elementen und Kind-Elementen
        (denke an binäre Bäume, um sich diese Relationen vorzustellen).
        Aber auch ohne Verschachtelung, Inhalt von einem HTML-Tag ist immer ein Kind 
        von dem entsprechenden HTML-Element.
        Auch wenn es nur ein Text/String ist.

SYNTAX VON ATTRIBUTEN

        Durch Hinzufügen eines HTML-Tags in einer HTML-Datei wird vom Browser in seinem Speicher 
        ein HTML-Element angelegt. 
        Das ist ein komplexes Objekt mit viiiiiiielen Eigenschaften,
        wie z.B. Id-Bezeichner, Höhe, Breite, ob es draggable ist, Relation zu anderen Tags u.v.m.
        Viele von diesen Eigenschaften bleiben einfach leer. 
        Sie sind dafür da, dem Web-Entwickler die Möglichkeit zu geben, 
        diese Eigenschaften mit eigenen Werten zu befüllen.
        Einige von diesen Eigenschaften bekommen sofort die Werte, 
        die einem HTML-Element von Anfang an eigen sind. 
        Wie z.B. die Schriftgröße bei den Überschriften erster Ordnung (h1) - 
        sie ist zweimal größer als die bei normalen Text-Absätzen (Paragraphen).
        Die Eigenschaften von HTML-Elementen können ihre Werte z.B. über HTML-Attribute bekommen.

SYNTAX UND FORMAT VON ATTRIBUTWERTEN

        Die meisten Attribute akzeptieren nur eine Auswahl an vordefinierten Werten
        oder nur bestimmtes Format von Daten.
        Z.B. `disabled` akzeptiert entweder `true` oder `false`, nichts Weiteres.
        Attribute `href` und `src` erwarten eine URL. 
        Viele anderen Attribute wie z.B. Attribute `alt`, akzeptieren beliebige Strings.
Anchor link 
        Bei true/false-Attributen ist die Angabe vom Wert seit HTML5 optional.
        Wenn das Attribut überhaupt angegeben ist, dann ist es automatisch auf `true` gesetzt.

IDENTIFIER: ID, CLASS & NAME

        HTML ist keine Programmiersprache, es ist unmöglich, eine HTML-Variable anzulegen.
        Aber mit Hilfe vom globalen Attribut `id` kann man ein bestimmtes HTML-Element unter vielen anderen identifizieren.
        Mit dem Attribut `class` wird eine Gruppe von Elementen identifiziert, 
        für die der Web-Entwickler gemeinsame CSS-Stile oder JS-Funktionalität vorgesehen hat.
        Mit dem Attribut `name` ist beides möglich. 
        Zusätzlich zu `id` und `class` wird `name` bei Formular-Elementen eingesetzt.
        Generell akzeptieren diese drei Attribute beliebige Zeichenkombinationen, außer Leerzeichen

p mit ID

p mit Klasse myClass

div mit Klasse myClass
        In HTML4 gab es strengere Regeln für die ID-Identifier. 
        Wenn Du an einem Projekt arbeitest, 
        wo die Kompatibilität mit HTML4 beibehalten werden soll, 
        dann benutze nur ASCII Buchstaben, Zahlen, `_`, `-` und `.`
        Die Zahlen dürfen nicht als erstes Zeichen benutzt werden.
