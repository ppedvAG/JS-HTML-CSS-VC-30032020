Datalist & Autovervollständigung

            Mit Datalist kann man dem Benutzer schon verbereite Einträge anbieten.
            Seine eigenen Eingaben sind auch möglich.
            In der .NET-Welt heißt es 'ComboBox'. 
            Unterschied zu select-option-Kombination: hier kann der Benutzer noch 
            eigene Eingaben eintippen.
            Autocomplete - filtern beim Tippen.

fieldset

            Fieldset wird wie ein Rahmen für mehrere Inputs benutzt. 
            In Legend wird Beschreibung für dieses Fieldset geschrieben.
            Placeholder: grauer Hinweis direkt im Input-Feld 
            Attribut Value: Anfangswert von Input, den man aber überschreiben kann
            Hinweise bei email-Feld kommen automatisch vom Browser
            Attr Title: Tooltip bei Mouseover
            Attr Pattern: RegExp, welche beim Validieren von der Eingabe eingesetzt werden.
            Validierungstexte in Feldern mit Pattern werden vom Browser ausgegeben. 
            In offiz. Docs werden Inputs innerhalb von Labels platziert.
            Über Attr. For bei Label und Id bei Input werden die beiden verbunden 
            (Klick auf Label fokusiert das Input-Feld).
            Buttons haben mehrere Types (submit, button, reset, menu). Default ist submit. 
            (Mehr dazu in der letzten Section)

Eingabe aus Input wird beim Klick auf Button in Output angezeigt

submitevent

            default-Wert vom Attribut type bei einem button innerhalb von form-Tag 
            ist submit!
            Beim event Submit werden alle Input-Felder vom Browser durchgelaufen und geprüft, 
            ob die Benutzereingaben den gestellten Bedingungen entsprechen. 
            Bei den Handlern von Submit-event sollte man die Funktion anders als 'checkValidity' nennen. 
            Wenn die Validity true ist, wird die Seite neu geladen.
            Ohne Attribut action bei form wird das Neuladen verhindert (für Testzwecke).

            mit ihren Input-Feldern aufgelistet.
