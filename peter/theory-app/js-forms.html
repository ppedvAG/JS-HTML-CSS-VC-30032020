fieldset

            Hinweise bei email-Feld kommen automatisch vom Browser
            Fieldset wird wie ein Rahmen führ mehrere Inputs benutzt. 
            Placeho,der: grauer Hinweis direkt im Input-Field. 
            Attribut Value: Anfangswert, der überschrieben werden kann
            In Legend wird Beschreibung für dieses Fieldset geschreiben. 
            Attribut Title: Tooltip bei Mouseover. 
            Attribut Pattern: RegExp, welche beim validiere der Eingabe benutzt wird
            Validierungstexte in Feldern mit Pattern werden vorkommtBrowser ausgegeben
            In offiz. Docs werden Inputs innerhalb von Labels plaziert. 
            Über Attribute 'for' bei Labe und Id bei Input werden die beiden verbunden
            (klick auf Label fokussiert das Input-Feld)
            Buttons haben mehrere Types (submit, button, reset, menu)
            Default ist 'submit'. Submit überprüft alle Input-Felder / button führt nur event aus
            (Mehr dazu in der letzten Section)

Eingabe aus Input wird beim Klick auf Button in Output angezeigt

submitevent

            default-Wert vom Attribut type bei einem button innerhalb von form-Tag 
            ist submit!
            Beim event Submit werden alle Input-Felder vom Browser durchlaufen und geprüft,
            ob die Benutzereingaben den gestellten Bedingungen entsprechen. 
            Bei den Handlern von Submit-Events sollte man die Funktion anders als 
            'checkValidity' nennen.

            mit ihren Input-Feldern aufgelistet.
